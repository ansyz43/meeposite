## Слайд 1 — Проблема рынка

### Каждый дистрибьютор FitLine сталкивается с одним и тем же

| | Проблема | Последствия |
|---|---------|------------|
| ⏳ | **Одинаковые вопросы каждый день** | «Что такое NTC?», «Чем Basics отличается от Restorate?» — вы отвечаете по 20 раз |
| 🌙 | **Клиенты пишут когда удобно им** | Сообщение в 23:00, вы спите — утром человек уже ушёл к другому |
| 📉 | **Невозможно масштабироваться** | 50+ контактов, каждому нужна консультация — физически не успеть |
| 🧠 | **Сложно объяснить продукт** | Составы, патенты, технология NTC — не каждый партнёр это донесёт |
| 🆕 | **Новички буксуют** | Не знают продукт, боятся общаться — теряют первых клиентов в первую же неделю |

> **Результат:** потерянные продажи, выгорание, медленный рост команды.

## Слайд 2 — Решение

# Meepo — AI-ассистент, который никогда не спит

Персональный бот в **Telegram** и **ВКонтакте** на базе OpenAI GPT, обученный на проверенной базе знаний FitLine.

| Что делает | Как именно |
|-----------|-----------|
| **Консультирует** | Знает весь каталог FitLine: состав, показания, технологию NTC, программы здоровья |
| **Продаёт мягко** | 5-этапный фреймворк: потребность → решение → польза → социальное подтверждение → ссылка |
| **Работает от вашего имени** | «Привет! Я ассистент Анны» — ваше имя, ваш аватар, ваш личный бот |
| **Передаёт готового покупателя** | Когда клиент заинтересован — бот отправляет вашу персональную ссылку |
| **Отвечает мгновенно** | 2–5 секунд на ответ, 24/7, без выходных, без обеда |

## Слайд 5 — Искусственный интеллект внутри

### RAG-система + GPT = точные ответы по FitLine

```
 Сообщение клиента
       ↓
 Извлечение ключевых слов
       ↓
 Расширение синонимами (55+ маппингов)
       ↓
 Оценка релевантности каждого продукта
       ↓
 Топ-5 продуктов → в контекст для GPT
       ↓
 GPT формирует ответ по фреймворку продаж
       ↓
 Ответ клиенту (2–5 сек)
```

| Параметр | Значение |
|----------|---------|
| **Модель** | OpenAI GPT (последнее поколение) |
| **База знаний** | 25+ продуктов FitLine, программы здоровья, регистрация, бизнес-информация |
| **RAG-оптимизация** | Промпт 2–3K токенов вместо 15K — экономия 80% стоимости API |
| **Стиль** | Мягкие продажи через пользу, а не через характеристики |
| **Контекст** | Помнит последние 10 сообщений в диалоге |
| **Ограничения** | Не даёт медицинских советов, не обсуждает конкурентов, не выдумывает |
| **Надёжность** | Резервная модель при сбое, circuit breaker, 3 автоматических retry |

## Слайд 6 — Реферальная система

### Приглашайте → зарабатывайте

```
🏆  Вы (владелец бота)
 ├── 👤 Партнёр 1 → использует вашего бота со своей ссылкой
 ├── 👤 Партнёр 2 → зарегистрировался по вашей реферальной ссылке
 └── 👤 Партнёр 3 → вы получаете кэшбэк с его покупок на платформе
```

| Механика | Описание |
|----------|---------|
| **Партнёрская ссылка** | `t.me/ВашБот?start=ref_XXXXXX` — уникальная для каждого партнёра |
| **Реферальная сессия** | 12 часов: все покупки через ссылку партнёра идут ему |
| **Кредиты** | 5 кредитов на старте, 1 кредит = 1 реферальная сессия |
| **Каталог** | Партнёр выбирает бота из каталога и подключается в 1 клик |
| **Дерево рефералов** | Визуализация всей партнёрской сети в личном кабинете |
| **Кэшбэк** | Процент с покупок на платформе Meepo (подписки, кредиты) |

> **Важно:** кэшбэк начисляется с покупок на платформе Meepo, а не с продаж продукции FitLine.

## Слайд 12 — Надёжность и масштаб

### Инженерные решения для стабильной работы

| Механизм | Что делает |
|----------|-----------|
| **Резервная AI-модель** | При сбое основной — автоматическое переключение на fallback |
| **Circuit breaker** | 5 ошибок подряд → пауза 60 сек → автоматический сброс |
| **Retry с backoff** | 3 повтора с экспоненциальной задержкой (2с → 4с) |
| **Дедупликация** | LRU-кэш 5000 записей — сообщение не обработается дважды |
| **Health checks** | Docker healthcheck для каждого контейнера + /api/health endpoint |
| **Auto-restart** | Ботов перезапускает auto backoff, контейнеры — Docker restart policy |
| **Семафор** | 30 параллельных AI-запросов — защита от rate limit OpenAI |
| **Синхронизация** | Каждые 10 секунд подхватываются новые боты из БД |
